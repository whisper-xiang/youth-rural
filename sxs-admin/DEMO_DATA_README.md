# 三下乡活动管理系统演示数据初始化指南

## 概述

本系统提供完整的演示数据初始化脚本，确保前端每个页面每种状态都有完整的数据，方便系统演示和测试。

## 数据覆盖范围

### 1. 项目申报模块
- ✅ **草稿状态**：1个项目（可编辑）
- ✅ **待审核状态**：2个项目（不同学院）
- ✅ **学院已审**：1个项目（等待校级审核）
- ✅ **已立项**：2个项目（可进行实践）
- ✅ **已驳回**：2个项目（不同驳回原因）
- ✅ **已撤回**：1个项目（用户主动撤回）
- ✅ **已结项**：1个项目（完整流程）

### 2. 审批管理模块
- ✅ **待学院审核**：2个项目
- ✅ **待校级审核**：1个项目
- ✅ **审批记录**：完整的审批历史
- ✅ **权限控制**：不同角色看到不同项目

### 3. 项目工作台模块
- ✅ **进度记录**：多个项目的进度数据
- ✅ **成果材料**：图片、文档、视频等成果
- ✅ **结项申请**：完整的结项流程
- ✅ **评优情况**：项目评优数据

### 4. 评优管理模块
- ✅ **进行中评优**：1个活动（可评分）
- ✅ **已完成评优**：1个活动（有结果）
- ✅ **专家评分**：多个专家的评分记录

### 5. 通知公告模块
- ✅ **活动通知**：重要活动通知
- ✅ **指南文档**：申报指南等
- ✅ **表彰决定**：优秀项目表彰
- ✅ **工作提醒**：安全提醒等

### 6. 消息中心模块
- ✅ **审批通知**：项目状态变更通知
- ✅ **进度更新**：项目进度提醒
- ✅ **成果发布**：新成果通知

## 使用方法

### 方法一：使用npm脚本（推荐）

```bash
# 在后端目录下执行
npm run init-demo
```

### 方法二：直接执行脚本

```bash
# 在后端目录下执行
node scripts/init_demo_data.js
```

### 方法三：手动执行SQL

```bash
# 连接数据库后执行
mysql -u root -p sxs_db < database/demo_data_init.sql
```

## 环境要求

- Node.js >= 14.0.0
- MySQL >= 5.7.0
- 数据库：sxs_db
- 用户表数据已初始化（通过seed.sql）

## 数据关系说明

### 项目状态流转
```
草稿 → 待审核 → 学院已审 → 校级已审 → 已立项 → 已结项
        ↓           ↓           ↓
      已驳回      已驳回      已撤回
```

### 权限控制
- **学生**：只能看到自己参与的项目
- **教师**：可以看到自己指导的项目
- **学院管理员**：只能审批本学院项目
- **校级管理员**：可以审批所有项目
- **专家**：可以参与评优项目

## 注意事项

1. **数据备份**：执行脚本前建议备份现有数据
2. **权限检查**：确保数据库用户有足够的权限
3. **环境配置**：检查数据库连接配置是否正确
4. **重复执行**：脚本可以重复执行，会清理并重新初始化数据

## 故障排除

### 常见问题

1. **连接数据库失败**
   - 检查数据库服务是否启动
   - 检查连接配置是否正确
   - 检查用户权限是否足够

2. **SQL执行失败**
   - 检查数据库表结构是否存在
   - 检查外键约束是否正确
   - 查看具体错误信息

3. **数据验证失败**
   - 检查SQL脚本是否完整执行
   - 查看验证日志信息
   - 手动检查数据库数据
