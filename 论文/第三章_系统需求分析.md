# 第三章 系统需求分析

## 3.1 系统总体需求分析

### 3.1.1 系统概述

本系统是一个面向高校的大学生暑期"三下乡"活动管理系统，旨在实现活动管理的信息化、规范化、便捷化。系统采用微信小程序作为前端载体，为用户提供随时随地的访问能力。

### 3.1.2 用户角色分析

通过对"三下乡"活动管理流程的调研分析，本系统确定了五种用户角色：

**（1）学生（团队负责人）**

学生是活动的主要参与者，负责组建实践团队、提交项目申报、上传活动进度、提交实践成果。学生用户的主要需求包括：
- 查看活动通知和政策文件
- 在线提交项目申报材料
- 实时查看审批进度
- 上传活动进展记录
- 提交实践成果材料
- 查看评优结果

**（2）指导教师**

指导教师负责指导学生开展实践活动，对学生提交的进度进行点评。指导教师的主要需求包括：
- 查看所指导的项目信息
- 对学生进度进行评论指导
- 查看项目审批状态
- 查看学生提交的成果

**（3）学院管理员**

学院管理员负责本学院项目的初审工作，对符合条件的项目推荐至学校审批。学院管理员的主要需求包括：
- 查看本学院的项目申报
- 对项目进行审批（通过/驳回）
- 查看本学院项目的进展情况
- 统计本学院的活动数据

**（4）校级管理员**

校级管理员负责全校项目的终审工作，以及活动通知的发布。校级管理员的主要需求包括：
- 查看全校的项目申报
- 对学院推荐的项目进行终审
- 发布活动通知和政策文件
- 管理评优活动
- 查看全校活动统计数据

**（5）评审专家**

评审专家负责对实践成果进行评审打分。评审专家的主要需求包括：
- 查看待评审的项目成果
- 对项目进行多维度评分
- 填写评审意见

### 3.1.3 业务流程分析

"三下乡"活动的主要业务流程如下：

```
活动发布 → 项目申报 → 学院审批 → 校级审批 → 开展活动 → 进度上传 → 成果提交 → 评优评审
```

各环节说明：

1. **活动发布**：校级管理员发布活动通知，说明活动主题、时间、要求等
2. **项目申报**：学生组建团队，填写申报材料，选择指导教师
3. **学院审批**：学院管理员审核本学院项目，通过后推荐至学校
4. **校级审批**：校级管理员对学院推荐项目进行终审，批准立项
5. **开展活动**：获批团队按计划开展实践活动
6. **进度上传**：团队定期上传活动进展，指导教师进行点评
7. **成果提交**：活动结束后，团队提交实践成果材料
8. **评优评审**：专家对成果进行评审，评选优秀团队

## 3.4 系统可行性分析

| 技术领域 | 可行性分析 |

| 前端技术 | 微信小程序技术成熟，文档完善，开发工具完备 |
